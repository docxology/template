# Syntax Guide

Syntax conventions for manuscript and project documentation in
`template_advanced_literature_review`.

## Links

Use descriptive Markdown links. Prefer repository-root paths in commands and
prose, for example
`projects/templates/template_advanced_literature_review/src/multi_phase/search.py`.

## Citations And Cross-References

Use Pandoc citation/crossref syntax:

```markdown
Prior work is cited with [@smith2020]. See [@fig:hypothesis_dashboard] and [@tbl:hypothesis_scores].
```

Do not hardcode figure, table, or section numbers.

## Manuscript Tokens

Use `{{TOKEN_NAME}}` for values computed from config and generated artifacts.
The authoritative token list is the return value of
`src/manuscript/variables/compute.py::compute_variables()` plus each
`extractors/` module and the manuscript-token tests.

### Standard token families (shared with the single-term sibling)

| Family | Source |
| --- | --- |
| `SEARCH_TERM*`, `ENGINE_LIST`, `N_ENGINES` | `manuscript/config.yaml` → `project_config.search` |
| subfield counts | `subfield_classification.json` |
| figure counts | `output/figures/` and figure registry |

### Phase-aware token families (project-specific, `extractors/multi_phase.py`)

| Token | Source |
| --- | --- |
| `TOTAL_PHASES` | Number of phases in `phase_metadata.json` |
| `CORPUS_SIZE` | Total papers across all phases, from `phase_metadata.json` |
| `PHASE_{n}_NAME`, `PHASE_{n}_PAPERS`, `PHASE_{n}_QUERIES` | Per-phase name, final paper count, and query count (`n` = 1, 2, 3, ... in phase order) |
| `PHASE_{n}_LLM_FILTER_RATE` | Fraction of phase-`n` papers removed by LLM filtering, when the phase's deterministic-filtered count is nonzero |
| `CROSS_PHASE_OVERLAP_PCT` | Average Jaccard similarity across the phase-overlap matrix |
| `CROSS_PHASE_CITATION_RATE` | Citation-validation rate from `phase_metadata.json` → `citation_validation` |

### Hypothesis token families (`extractors/hypotheses.py`)

| Token | Source |
| --- | --- |
| `N_HYPOTHESES`, `HYPOTHESIS_LIST`, `HYPOTHESIS_TABLE` | `manuscript/config.yaml` → `project_config.hypothesis_definitions` |
| `TOTAL_ASSERTIONS`, `TOTAL_ASSERTIONS_RAW` | `assertion_summary.json` (knowledge-graph stage) |
| `{HID}_SUPPORT`, `{HID}_CONTRADICT`, `{HID}_NEUTRAL`, `{HID}_TOTAL` | Per-hypothesis assertion counts, `HID` = the config key (`H1`, `H2`, ...) |
| `ASSERTION_SUPPORT_PCT`, `ASSERTION_CONTRADICT_PCT` | Corpus-wide support/contradict percentages |

Before the knowledge-graph stage (03) has run, hypothesis tokens read
`pending` rather than raising — see `troubleshooting.md`.

Detect unresolved tokens before rendering:

```bash
rg "\{\{" projects/templates/template_advanced_literature_review/output/manuscript
```

## Figures

Figures are generated by `src/visualization/figure_runner.py` (symlinked from
the single-term sibling). Add figures by:

1. Creating or updating a plotting function in `src/visualization/` — edit it
   in `template_literature_meta_analysis`, not through this project's symlink.
2. Registering it through the figure runner.
3. Adding tests in the sibling project's `tests/visualization/`.
4. Referencing it from this project's manuscript prose with a Pandoc figure
   anchor, for example `{#fig:hypothesis_dashboard}`.
5. Updating `docs/output_inventory.md` when the artifact is part of the
   contract.

## Tables

Use Pandoc table captions below the table:

```markdown
| Hypothesis | Score |
| --- | ---: |
| H1 | +0.42 |

: Citation-weighted hypothesis scores. {#tbl:hypothesis_scores}
```

## Code Blocks

Always include a language tag:

```bash
uv run pytest projects/templates/template_advanced_literature_review/tests/ -q
```
