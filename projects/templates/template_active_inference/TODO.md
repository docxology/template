## Promotion rule

A future track or within-track capability becomes live only after all of these
exist in the repository and pass under the core pipeline:

| Requirement | Minimum proof before promotion |
| --- | --- |
| Producer | Configured analysis script or renderer listed in the project DAG |
| Artifact | Deterministic file under `output/data/`, `output/figures/`, or `output/reports/` |
| Manuscript consumer | Bound IMRAD fragment or generated table that cites the artifact |
| Typed claim evidence | Claim-ledger row with field/predicate/tolerance when applicable |
| Semantic restriction | Certificate field that can detect disagreement or stale output |
| Validation gate | `validate_outputs`, `validate_manuscript`, Lean build, or project test |
| Negative control | Test that mutates the artifact/claim/config and proves the gate fails |
