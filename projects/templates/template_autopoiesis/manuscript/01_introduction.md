## Introduction

### The adversarial framing

Project generation systems routinely make three claims that are hard to verify:

1. **Completeness** — the generated project contains all necessary components.
2. **Determinism** — the same inputs deterministically produce identical outputs.
3. **Traceability** — every generated byte can be traced to its specification.

`template_autopoiesis` was designed to make each claim structurally verifiable,
not merely asserted in prose.

### Three defences

**Defence A — Grammar-anchored generation.**  
Every selection is derived via SHA-256 digest from the seed, slot name, ordinal,
and option list.  Changing any input changes the selection deterministically.

**Defence B — Tree-hash provenance.**  
Every materialized child records a Merkle-style tree hash computed from all
generated file contents.  The `verify` command recomputes this hash from live
files and flags any post-generation modification.
