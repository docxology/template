It must run in any Python environment without the template\'s pipeline installed. All cross-cutting concerns (logging, reporting, validation, rendering) are delegated to `infrastructure/` and called from `scripts/`. This keeps the code testable with zero mocks and the math reusable outside the pipeline.\n\n### What is the "thin orchestrator', 'pattern?\n\nFiles in `scripts/` should not implement mathematical operations. They should:\n- Import pure functions from `src/`.\n- Run experiment loops (calling those functions).\n- Delegate side-effects (file I/O, logging, progress bars) to `infrastructure/`.\n- Write outputs to `output/`.\n\nIf you find yourself writing a gradient update or a quadratic evaluation inside `scripts/`, move it to `src/` and add a test.\n\n### Why are mocks forbidden?\n\nMocking tests that a function *was called* instead of that it produced the *correct result*. Pure functions can be tested with real inputs and expected outputs. Mocks give a false sense of correctness and break the reproducibility guarantee.\n\n---\n\n## Testing and Coverage\n\n### Why 90% coverage? Can I lower it?\n\nThe coverage gate ensures `src/` logic is thoroughly exercised. `code_project` targets ~99%. Lowering the gate compromises the exemplar\'s authority. If coverage drops, it signals missing tests — add them, don\'t lower the gate.\n\n### My new function lowered coverage. What now?\n\nWrite unit tests for the new function before or immediately after adding it. Follow the existing test classes in `tests/test_optimizer.py`. Use fixed arrays and assert mathematical properties (convergence, gradient accuracy, shape checks).\n\n### Do I need to test `scripts/`?\n\n`scripts/` is tested via integration tests that run when `infrastructure` is available. Those tests call the main function (e.g., `run_stability_analysis()`) and assert it produces output files. They are marked to skip gracefully if `infrastructure` is not installed. See [`../../docs/testing_philosophy.md`](testing_philosophy.md).\n\n---\n\n## Manuscript and Rendering\n\n### How do I add a new figure?\n\n1.
