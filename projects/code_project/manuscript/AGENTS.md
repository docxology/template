# manuscript/ - Research Manuscript

## Overview

The `manuscript/` directory contains markdown-formatted research manuscript sections. These files document the optimization research, mathematical formulations, and experimental results. The manuscript is processed by the template's PDF rendering pipeline to generate professional documents.

## Key Concepts

- **Research manuscript** sections in markdown format
- **Mathematical notation** using LaTeX-style equations
- **Cross-references** between sections and figures
- **Professional formatting** for academic publication

## Directory Structure

```
manuscript/
├── 01_abstract.md               # Research overview and contributions
├── 02_methodology.md            # Mathematical framework and algorithms
├── 03_results.md                 # Experimental results and analysis
├── 04_conclusion.md              # Summary and future directions
├── references.bib                # Bibliography in BibTeX format
├── AGENTS.md                    # This technical documentation
└── README.md                    # Quick reference
```

## Installation/Setup

The manuscript uses standard markdown with LaTeX math extensions. No special setup required beyond the template's pandoc and LaTeX installation.

## Usage Examples

### Edit Manuscript Sections

```bash
# Edit individual sections
vim manuscript/02_methodology.md
vim manuscript/03_results.md

# View rendered PDF
python3 ../../scripts/03_render_pdf.py
open ../../output/pdf/project_combined.pdf
```

### Cross-Reference Figures

Figures generated by scripts are automatically referenced:

```
See the convergence analysis in Figure \ref{fig:convergence}.

![Gradient descent convergence for different step sizes](../output/figures/convergence_plot.png){#fig:convergence}
```

Note: This example uses the actual convergence_plot.png generated by the analysis scripts.

## Configuration

Manuscript sections follow academic paper structure:

- **01_abstract.md**: Concise summary (200-300 words)
- **02_methodology.md**: Mathematical formulations and algorithms
- **03_results.md**: Experimental results with figure references
- **04_conclusion.md**: Summary and implications

## Testing

Manuscript quality is validated by the template's validation system:

```bash
# Validate markdown structure
python3 -m infrastructure.validation.cli markdown .

# Check cross-references
python3 -m infrastructure.validation.cli integrity ../../output/
```

## Troubleshooting

### Common Issues

- **Missing figures**: Ensure scripts have been run to generate figures
- **Broken references**: Check figure labels match references
- **LaTeX errors**: Validate mathematical notation syntax

### Debug Tips

Preview individual sections:
```bash
# Render single section
python3 ../../scripts/03_render_pdf.py --section 02_methodology
```

## Best Practices

- **Mathematical notation**: Use LaTeX-style equations for clarity
- **Figure references**: Always label figures for cross-referencing
- **Consistent formatting**: Follow academic writing conventions
- **Version control**: Commit manuscript changes regularly
