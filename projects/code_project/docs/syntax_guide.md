# Syntax Guide

This document defines the syntax conventions for documentation and manuscript content inside the `code_project` exemplar. Rules 1–5 are mandatory constraints; sections 6–8 are reference material for common operations.

---

## 1. Markdown Links

Hyperlinks must be informative. Never use placeholder text.

- **BAD**: [this link](https://github.com/docxology/template) to see the template.
- **GOOD**: See the [Research Project Template](https://github.com/docxology/template).

For internal cross-references inside `manuscript/`, prefer relative paths:
- **BAD**: See `https://github.com/docxology/template/blob/main/projects/code_project/src/optimizer.py`
- **GOOD**: See [`projects/code_project/src/optimizer.py`](https://github.com/docxology/template/blob/main/projects/code_project/src/optimizer.py)

---

## 2. LaTeX Cross-References

Inside `manuscript/` files, use `\ref{}` for explicit cross-referencing. Never hardcode figure or table numbers.

- **BAD**: See Table 1 or Figure 2.
- **GOOD**: See Table `\ref{tab:opt_results}` or Figure `\ref{fig:convergence}`.

## 3. Variable Injection (Madlibs)

When specifying numeric results in the manuscript, use the `{{VARIABLE_NAME}}` syntax. Values are hydrated dynamically by `scripts/z_generate_manuscript_variables.py`. Never hardcode a number that will change when the experiment parameters change.

- **BAD**: The algorithm took 165 iterations.
- **GOOD**: The algorithm took `{{RESULT_MAX_ITERATIONS}}` iterations.

### Adding a New Variable

1. Define the variable in `scripts/z_generate_manuscript_variables.py` — add a key/value pair to the `variables` dict in `build_variables()`.
2. Verify it produces a non-empty string: `python scripts/z_generate_manuscript_variables.py && python -c "import json; d=json.load(open('projects/code_project/output/data/manuscript_variables.json')); print(d['MY_TOKEN'])"`.
3. Reference it in a manuscript `.md` file as `{{MY_TOKEN}}`.

### Detecting Unresolved Tokens

If a token remains unresolved, the literal `{{TOKEN_NAME}}` will appear in the rendered PDF. Detect before rendering:

```bash
grep -r "{{" projects/code_project/output/manuscript/ 2>/dev/null \
  && echo "UNRESOLVED TOKENS FOUND" || echo "All tokens resolved"
```

## 4. Code Blocks

Always tag code blocks with their language identifier. This is required for Pandoc syntax highlighting in the PDF.

```python
def example() -> bool:
    return True
```

For shell commands:
```bash
uv run pytest projects/code_project/tests/ -v
```

For inline code referencing file paths, use single backticks: `projects/code_project/src/optimizer.py`

---

## 5. Table Captions (Pandoc)

When writing tables in the manuscript, place the caption below the table using Pandoc syntax. The `\label{}` must be inside the caption line:

```markdown
| Step Size | Iterations | Converged |
|-----------|------------|-----------|
| 0.01      | 1000       | No        |
| 1.00      | 1          | Yes       |

: Gradient descent outcomes per step size. \label{tab:opt_results}
```

Do not use `Table:` prefix — Pandoc infers the type from placement. Do not hardcode the table number in text; always use `\ref{tab:opt_results}`.

## 6. Adding a New Figure

To add a figure that appears in `03_results.md`:

1. Add the generator function to `scripts/optimization_analysis.py` following the existing pattern (write PNG to `projects/code_project/output/figures/`).
2. Add the Pandoc image reference in `03_results.md`:
   ```markdown
   ![Caption text describing the figure.](../output/figures/new_figure.png){#fig:new_label}
   ```
3. Reference in text: `Figure \ref{fig:new_label}`.
4. Update `output/AGENTS.md` to document the new file in the regeneration table.
5. Re-run the full pipeline to verify the figure appears in the PDF.

## 7. LaTeX Math in Manuscript

Use `$...$` for inline math and `$$...$$` for display equations. Pandoc converts these to LaTeX automatically.

For numbered equations (referenced with `\ref{}`), use LaTeX `equation` environment inside a fenced code block:

```latex
\begin{equation}
x_{k+1} = x_k - \alpha \nabla f(x_k)
\label{eq:gradient_update}
\end{equation}
```

Reference in text: `Equation \ref{eq:gradient_update}`.
