# Output Directory Schema

Layout and conventions for the generated `output/` directory. All contents are **disposable** — regenerated by the pipeline on every run.

## Directory Structure

```text
output/
├── .checkpoints/       # Pipeline checkpoint state (internal use)
├── citations/          # Generated citation files
│   ├── *.bib           #   BibTeX format
│   ├── *.txt           #   APA/MLA text format
│   └── *.json          #   Citation metadata
├── data/               # Numerical results
│   ├── *.csv           #   Tabular data (optimization results)
│   └── *.npz           #   NumPy compressed arrays
├── docs/               # Generated API documentation
│   └── *.md            #   Auto-generated module docs
├── figures/            # Publication-quality plots
│   ├── convergence_plot.png
│   ├── stability_analysis.png
│   ├── benchmark_comparison.png
│   └── *.png           #   300 DPI, tight bbox
├── llm/                # LLM review outputs (optional)
│   ├── *.md            #   Review summaries
│   └── *.json          #   Structured review data
├── logs/               # Pipeline execution logs
│   └── pipeline.log    #   Timestamped log output
├── pdf/                # Rendered manuscripts
│   ├── *_combined.pdf  #   Full manuscript
│   ├── 00_abstract.pdf #   Per-section PDFs
│   └── *.tex           #   Intermediate LaTeX
├── reports/            # Analysis reports
│   ├── *.json          #   Stability/benchmark reports
│   ├── *.html          #   Dashboard (analysis_dashboard.html)
│   └── *.md            #   Markdown summaries
├── simulations/        # Simulation outputs (if applicable)
├── slides/             # Generated presentation slides
│   └── *.html          #   HTML slide decks
├── tex/                # Raw LaTeX intermediates
│   └── *.tex           #   Pre-compiled TeX source
└── web/                # Web-format outputs
    ├── *.html           #   HTML manuscript versions
    └── assets/          #   Supporting web assets
```

## File Naming Conventions

| Pattern | Example | Used In |
|---------|---------|---------|
| `{name}_combined.pdf` | `code_project_combined.pdf` | `pdf/` |
| `{NN}_{section}.pdf` | `00_abstract.pdf` | `pdf/` |
| `{analysis_type}.png` | `convergence_plot.png` | `figures/` |
| `{data_type}.csv` | `optimization_results.csv` | `data/` |
| `{report_type}.json` | `stability_analysis.json` | `reports/` |
| `analysis_dashboard.html` | — | `reports/` |
| `pipeline.log` | — | `logs/` |

## Regeneration

All output files are regenerated by running:

```bash
# Full pipeline (cleans output first)
./run.sh --pipeline

# Individual stages
python3 scripts/optimization_analysis.py  # figures/, data/, reports/
python3 scripts/03_render_pdf.py          # pdf/, tex/
```

## Disposability Policy

> **Every file in `output/` is disposable.** The pipeline regenerates everything from source. Never manually edit output files — they will be overwritten.

- `output/` is excluded from version control (`.gitignore`)
- Pipeline stage 0 cleans the output directory before each run
- `.checkpoints/` tracks pipeline resume state and is also disposable
