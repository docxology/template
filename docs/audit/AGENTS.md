# audit/ - Audit Reports Documentation

## Overview

This directory stores automated audit reports generated by repository-wide validation tools. These reports identify documentation issues, broken links, and quality problems across the codebase.

**Living rosters** of `projects/` names belong in [_generated/active_projects.md](../_generated/active_projects.md), not in these reports. Report line references are **point-in-time snapshots** from their generation run.

## Purpose

The audit directory serves as a repository for:

- Filepath validation reports
- Link integrity checks
- Cross-reference validation
- Documentation quality assessments
- Repository-wide issue tracking

## File Types

### FILEPATH_AUDIT_REPORT.md / filepath-audit-report.md

Both names refer to the **same report shape** (markdown or JSON via `--format`). The script default is `docs/audit/FILEPATH_AUDIT_REPORT.md`; use `--output` to write elsewhere (for example `filepath-audit-report.md`).

Report contents include:

- **Link Issues**: Broken references, missing files, invalid paths
- **Quality Issues**: Documentation gaps, formatting problems, missing content
- **Executive Summary**: Issue counts by category
- **Detailed Findings**: Line-by-line issue reports with file paths

### documentation-review-report.md

Documentation completeness review report containing:

- **File Coverage**: Analysis of documented vs undocumented files
- **Content Quality**: Review of documentation accuracy and depth
- **Cross-Reference Integrity**: Validation of internal links

### documentation-review-summary.md

Executive summary of documentation review:

- **High-Level Findings**: Key observations and statistics
- **Priority Recommendations**: Action items ranked by importance

## Report Generation

**CLI (preferred):** [`scripts/audit_filepaths.py`](../../scripts/audit_filepaths.py) — see [README.md](README.md).

**Programmatic:**

```python
from pathlib import Path

from infrastructure.validation.repo.audit_orchestrator import (
    generate_audit_report,
    run_comprehensive_audit,
)

repo_root = Path(".")
scan_results = run_comprehensive_audit(
    repo_root,
    verbose=False,
    include_code_validation=True,
    include_directory_validation=True,
    include_import_validation=True,
    include_placeholder_validation=True,
)
report_md = generate_audit_report(scan_results, "markdown")
output_path = Path("docs/audit/FILEPATH_AUDIT_REPORT.md")
output_path.write_text(report_md, encoding="utf-8")
```

Public symbols are re-exported from [`infrastructure.validation`](../../infrastructure/validation/__init__.py) for convenience.

## Report Structure

### Executive Summary

- Total issues found
- Issues by category (links, quality)
- Scan duration and file count

### Link Issues Section

- Broken file references
- Missing target files
- Invalid path formats
- Cross-reference failures

### Quality Issues Section

- Documentation gaps
- Formatting problems
- Missing required sections
- Incomplete content

## Integration

Audit reports integrate with:

- **Validation Pipeline**: `scripts/04_validate_output.py` (output validation stage)
- **Implementation**: [`infrastructure/validation/repo/audit_orchestrator.py`](../../infrastructure/validation/repo/audit_orchestrator.py)
- **Orchestrator script**: [`scripts/audit_filepaths.py`](../../scripts/audit_filepaths.py)

## Maintenance

- Regenerate reports with `uv run python scripts/audit_filepaths.py` when validating docs against the current tree
- Historical copies may be kept under alternate filenames; compare dates in report headers
- Tune false positives in [`infrastructure/validation/repo/known_exceptions.py`](../../infrastructure/validation/repo/known_exceptions.py) when justified
